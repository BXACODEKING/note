## 🔄 完整流程说明

  

### 1. **平台下发 SetParameterValues**

```xml

<SetParameterValues>

  <ParameterList>

    <ParameterValueStruct>

      <Name>InternetGatewayDevice.IPPingDiagnostics.Interface</Name>

      <Value>Device.WAN.Interface.1</Value>

    </ParameterValueStruct>

    <ParameterValueStruct>

      <Name>InternetGatewayDevice.IPPingDiagnostics.Host</Name>

      <Value>8.8.8.8</Value>

    </ParameterValueStruct>

    <ParameterValueStruct>

      <Name>InternetGatewayDevice.IPPingDiagnostics.NumberOfRepetitions</Name>

      <Value>5</Value>

    </ParameterValueStruct>

    <ParameterValueStruct>

      <Name>InternetGatewayDevice.IPPingDiagnostics.Timeout</Name>

      <Value>1000</Value>

    </ParameterValueStruct>

    <ParameterValueStruct>

      <Name>InternetGatewayDevice.IPPingDiagnostics.DataBlockSize</Name>

      <Value>64</Value>

    </ParameterValueStruct>

    <ParameterValueStruct>

      <Name>InternetGatewayDevice.IPPingDiagnostics.DiagnosticsState</Name>

      <Value>Requested</Value>  <!-- 最后设置，触发诊断 -->

    </ParameterValueStruct>

  </ParameterList>

</SetParameterValues>

```

  

### 2. **终端处理流程**

  

```

easycwmpd (接收 SetParameterValues)

    ↓

调用 external script (easycwmp)

    ↓

easycwmp 接收 JSON 格式的 SET 命令

    ↓

cwmpfunc_operate() 遍历 cwmpnode_tr 数组

    ↓

匹配到 IPPingDiagnostics 参数，调用 cwmpset_Diagnostics()

    ↓

cwmpset_Diagnostics() 保存参数到 /tmp/.cwmp.IPPingDiagnostics.json

    ↓

当 DiagnosticsState = "Requested" 时

    ↓

启动 /usr/sbin/cwmpDiagnostics IPPingDiagnostics 进程

    ↓

cwmpDiagnostics 读取 JSON 配置，执行 ping 命令

    ↓

解析 ping 结果，更新 SuccessCount、FailureCount、MinimumResponseTime 等

    ↓

设置 DiagnosticsState = "Complete" 或 "Error_XXX"

    ↓

执行 ubus call tr069 inform '{"event":"8 DIAGNOSTICS COMPLETE"}'

    ↓

easycwmpd 接收 ubus 调用，触发事件上报

    ↓

发送 Inform 消息，EventCode = "8 DIAGNOSTICS COMPLETE"

```

  

### 3. **平台查询结果**

平台接收到 "8 DIAGNOSTICS COMPLETE" 事件后，发送 GetParameterValues 查询结果：


## 🐛 常见问题排查

  

### 问题 1：SET 命令执行后没有启动诊断

**排查步骤：**

```bash

# 1. 检查参数是否注册

echo '{"command":"get","class":"value","parameter":"InternetGatewayDevice.IPPingDiagnostics."}' | /usr/sbin/easycwmp --json-input

  

# 2. 检查 JSON 文件是否生成

ls -l /tmp/.cwmp.IPPingDiagnostics.json

  

# 3. 检查 cwmpDiagnostics 是否存在

ls -l /usr/sbin/cwmpDiagnostics

  

# 4. 手动测试诊断进程

/usr/sbin/cwmpDiagnostics IPPingDiagnostics

```

  

### 问题 2：诊断完成但没有上报事件

**排查步骤：**

```bash

# 1. 检查 ubus 服务

ubus list | grep tr069

  

# 2. 手动触发事件

ubus call tr069 inform '{"event":"8 DIAGNOSTICS COMPLETE"}'

  

# 3. 查看 easycwmpd 日志

logread | grep "triggered ubus inform"

```

  

### 问题 3：ping 测试失败

**排查步骤：**

```bash

# 1. 检查网络接口

ifconfig

  

# 2. 手动 ping 测试

ping -I eth1 -c 5 -s 64 -w 1 8.8.8.8

  

# 3. 查看诊断结果文件

cat /tmp/.cwmp.IPPingDiagnostics.result

```
