# 远程升级 Inform 失败排查思路

  

## 问题现象

  

从日志看到：

1. `[Event Queue] Will execute: /mnt/update/easycwmp (code=-1)` - 事件队列异常

2. `sending Inform failed` - Inform 发送失败

3. `can't remove file /tmp/easycwmp_cookies` - Cookie 文件无法删除

4. 不断重试（7秒、15秒、30秒、60秒、120秒间隔）

  

## 排查步骤（按优先级）

  

### 步骤1：检查事件队列状态

  

**问题点：** `code=-1` 表示事件队列为空或异常

  

**排查方法：**

```bash

# 查看日志中的事件队列信息

grep "Event Queue" /var/log/messages

  

# 应该看到类似：

# [Event Queue] Current events in queue:

#   - 2 PERIODIC (code=2)

```

  

**可能原因：**

- 事件队列为空（没有待上报的事件）

- 事件队列被意外清空

- 事件添加失败

  

**修复建议：**

- 检查事件添加逻辑（`cwmp_add_event()`）

- 确认远程升级时是否正确添加了事件

  

---

  

### 步骤2：检查 Inform XML 生成

  

**问题点：** Inform 消息可能生成失败

  

**排查方法：**

```bash

# 查看保存的 Inform XML 文件

cat /tmp/easycwmp_inform_send.xml

  

# 检查文件是否存在

ls -l /tmp/easycwmp_inform_send.xml

```

  

**可能原因：**

- XML 生成失败（`xml_prepare_inform_message()` 返回 -1）

- 参数过多导致 XML 过大

- 内存不足

  

**修复建议：**

- 查看日志中的 "Inform xml message creating failed"

- 检查 `/tmp/easycwmp_inform_send.xml` 文件内容

  

---

  

### 步骤3：检查 HTTP 连接和错误信息

  

**问题点：** `sending Inform failed` 的具体原因

  

**排查方法：**

```bash

# 查看详细的 HTTP 错误信息（已添加日志）

grep "LibCurl Error\|curl_res=\|http_code=" /var/log/messages

  

# 检查网络连接

ping acs.jingxuntech.com

telnet acs.jingxuntech.com 8443

```

  

**可能原因：**

1. **网络连接问题**

   - DNS 解析失败

   - 网络不通

   - 防火墙拦截

  

2. **HTTPS/SSL 问题**

   - 证书验证失败

   - SSL 握手失败

   - 证书过期

  

3. **HTTP 状态码错误**

   - 返回 401（认证失败）

   - 返回 403（禁止访问）

   - 返回 500（服务器错误）

   - 返回其他非 200/204 状态码

  

4. **CURL 错误**

   - `CURLE_COULDNT_CONNECT` (7) - 无法连接服务器

   - `CURLE_SSL_CONNECT_ERROR` (35) - SSL 连接错误

   - `CURLE_OPERATION_TIMEDOUT` (28) - 超时

  

**修复建议：**

- 查看新增的详细错误日志（curl_res、http_code）

- 检查 ACS URL 配置是否正确

- 检查 SSL 证书配置

  

---

  

### 步骤4：检查 Cookie 文件问题

  

**问题点：** `can't remove file /tmp/easycwmp_cookies`

  

**排查方法：**

```bash

# 检查文件状态

ls -l /tmp/easycwmp_cookies

lsof /tmp/easycwmp_cookies  # 查看是否有进程占用

  

# 检查文件系统

df -h /tmp

mount | grep /tmp

```

  

**可能原因：**

- 文件被其他进程锁定

- 文件系统只读

- 权限问题

- 文件不存在（但代码尝试删除）

  

**修复建议：**

- 检查文件系统状态

- 检查是否有其他进程占用文件

- 这个错误通常不影响功能，但需要修复

  

---

  

### 步骤5：检查远程升级相关逻辑

  

**问题点：** 进入 `/mnt/update/easycwmp` 逻辑但失败

  

**排查方法：**

```bash

# 检查升级脚本是否存在

ls -l /mnt/update/easycwmp

  

# 检查升级脚本权限

file /mnt/update/easycwmp

  

# 查看升级相关的日志

grep "download\|upgrade\|firmware" /var/log/messages

```

  

**可能原因：**

- 升级脚本不存在或无法执行

- 升级过程中网络中断

- 升级脚本执行失败

  

**修复建议：**

- 确认 `/mnt/update/easycwmp` 脚本存在且可执行

- 检查升级脚本的日志输出
