## 命名不一致导致配置不生效的原因

### 核心问题：CLI 命令匹配使用精确字符串比较

command.cLines 1554-1558

                        if (strcmp (command, str) == 0)

                        {

                            match_type = exact_match;

                            matched++;

                        }

cmd_filter_by_string 使用 strcmp() 进行精确匹配。命令名必须完全一致才能匹配成功。

### 问题场景分析

#### 修复前的情况：

1. cli.conf 文件中的内容：
    
       interface modem sim1.1
    
       freqBand 78          ← 驼峰命名
    

2. CLI 命令定义（修复前）：
    
       DEFUN(..., "freq_band WORD", ...)  ← 下划线命名
    

3. 执行流程：
    
       config_from_file 读取: "freqBand 78"
    
       ↓
    
       cmd_execute_command_strict 尝试匹配
    
       ↓
    
       strcmp("freqBand", "freq_band") != 0  ← 匹配失败！
    
       ↓
    
       matched_count = 0
    
       ↓
    
       返回 CMD_ERR_NO_MATCH
    
       ↓
    
       命令不执行，参数不加载到内存
    

### 修复后的情况：

1. CLI 命令定义（修复后）：
    
    modem_conf.cLine 1408
    
            "freqBand WORD",
    

2. 执行流程：
    
       config_from_file 读取: "freqBand 78"
    
       ↓
    
       cmd_execute_command_strict 尝试匹配
    
       ↓
    
       strcmp("freqBand", "freqBand") == 0  ← 匹配成功！
    
       ↓
    
       matched_count = 1
    
       ↓
    
       执行 config_ppp_freq_band_lock 函数
    
       ↓
    
       g_modem_if->freqBand = "78"  ← 参数加载到内存
    

## 为什么必须精确匹配？

CLI 框架的设计原则：

1. 精确性：避免误匹配（如 freqBand 不应匹配 freq_band）

2. 性能：字符串精确比较效率高

3. 可预测性：用户输入与命令定义必须一致

## 总结

|场景|cli.conf|命令定义|匹配结果|配置是否生效|
|---|---|---|---|---|
|修复前|freqBand 78|freq_band WORD|失败|否|
|修复后|freqBand 78|freqBand WORD|成功|是|

结论：命名不一致会导致 strcmp() 匹配失败，命令不执行，参数不会加载到内存，因此配置不生效。修复后，命令名与配置文件中的名称一致，匹配成功，配置正常生效。
